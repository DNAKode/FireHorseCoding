# KodePorter

**KodePorter is a map and control system for creating, understanding, verifying, and continuously preserving software ports.**

In practical terms, KodePorter keeps a software port alive: it records what must correspond, what may differ, how changes should propagate, and what evidence is required before the target is trusted.

Start with [CHARTER.md](CHARTER.md). It is the governing statement of the project's purpose, domain model, relationship with Gneiss, constitutional principles, first implementation, and evolution discipline.

The longer [brainstorming and project-framing notes](docs/brainstorming-and-project-framing.md) preserve the initial exploration and research directions. The `transcripts/` directory preserves the original discussion trail.

KodePorter is the first deliberately co-developed realization of [Gneiss](../Gneiss/CHARTER.md). Gneiss supplies the reusable epistemic substrate; KodePorter supplies the porting domain and workflow.
